# Hajimi Ref 更新说明

## 2026-02-09 周一 - 组成员判定与智能整理优化

### 下午 17:32 - 修复 & 优化：组成员判定逻辑与智能整理后组框自适应
- **修复问题1（Win11）**：图片不在组框内却随组移动 / 图片在组框内却不随组移动
  - 根本原因：组成员关系通过 `group_id` 属性管理（逻辑关系），而非位置关系。当用户手动调整组框大小（resize）缩小时，`check_images_in_group_bounds` 只检查新图片进入组，**不检查现有成员是否仍在框内**。导致图片被拖出框外后 `group_id` 仍然指向该组，跟着组一起移动
  - 修复方案：重写 `check_images_in_group_bounds`，在拉入新图片之前，先遍历现有成员检查中心点是否仍在组框内，不在则自动移除
- **修复问题2（Win11）**：`check_image_out_of_group` 使用 `intersects`（只要有一点重叠就不移除）与 `check_images_in_group_bounds` 使用中心点判定标准不一致
  - 修复方案：统一为中心点判定——图片中心不在组框内即移出组
- **优化（Win11）**：组内智能整理后自动调整组框大小
  - 之前 `organize_items` 完成后不更新组边界，导致图片溢出组框
  - 修复方案：在 `organize_items` 末尾收集被整理图片所属的组ID，调用 `update_group_bounds` 自动调整组框大小以包含所有成员图片
- **影响文件**：
  - Windows: `Views/MainWindow.py`

## 2026-02-09 周一 - 双击编辑组名称：彻底修复（双端）

### 下午 17:24 - 修复：macOS 端双击组名称无法编辑
- **问题**：macOS 端（SwiftUI）双击组名称标签无法触发编辑
- **根本原因**：名称标签使用 `.offset(y: -group.fontSize - 12)` 定位到组框上方。SwiftUI 中 `.offset()` **仅改变视觉位置，不改变 hit testing 区域**。用户双击看到的标签位置时，点击事件实际命中的是原始位置（组框内部），而非标签的视觉位置。同时 `GroupView` 上的 `DragGesture` 和 `onTapGesture` 也会抢占名称标签上的双击手势
- **修复方案**：
  - 将名称标签从 ZStack 内部移出，改用 `.overlay(alignment: .topLeading)` 方式定位
  - 使用 `.alignmentGuide(.top) { d in d[.bottom] + 4 }` 将标签正确推到组框上方，且 hit testing 区域与视觉位置一致
  - 名称标签上独立绑定 `onTapGesture(count: 2)` 和 `onTapGesture(count: 1)`，不受 GroupView 上的 DragGesture 干扰
- **影响文件**：
  - macOS: `Views/CanvasView.swift`（`GroupView` 结构体重构名称标签定位方式）

### 下午 17:19 - 修复：Win11 端双击组名称无法编辑
- **问题**：Win11 端（Qt）双击组名称标签始终无法触发编辑
- **根本原因**：`GroupItem` 的 `zValue` 为 `-100`（在图片下方），且 `RefView` 设置了 `RubberBandDrag` 模式。在此模式下，`QGraphicsView` 会优先将空白区域的鼠标事件用于橡皮筋框选，根本不会将事件分发给底层的 `GroupItem`
- **修复方案**：在 `RefView`（QGraphicsView 层）新增 `mouseDoubleClickEvent` 方法，在 view 层主动拦截双击事件，将场景坐标转换为每个 `GroupItem` 的 item 局部坐标，检测是否落在 `_name_label_local_rect()` 区域内
- **影响文件**：
  - Windows: `Views/Canvas.py`（在 `RefView` 类中新增 `mouseDoubleClickEvent`）

## 2026-02-06 周四 - 组名称布局优化与双击编辑修复

### 下午 17:48 - 修复 & 优化：组名称交互与布局
- **修复问题（Win11）**：双击组名称无法快速编辑（彻底修复）
  - 根本原因：`QGraphicsRectItem` 的 `shape()` 方法默认只返回 `rect()` 区域的路径。而名称标签绘制在 `rect()` **外侧上方**，不在 `shape()` 范围内。Qt 事件分发系统要求点击位置必须落在 item 的 `shape()` 内才会将事件路由到该 item。因此名称标签区域的双击事件**永远无法到达** `GroupItem`
  - 修复方案：重写 `shape()` 方法，使用 `QPainterPath` 同时包含 `rect()` 和名称标签的区域。新增 `_name_label_local_rect()` 辅助方法，在 item 局部坐标系中计算名称标签的固定区域（不依赖 lod），供 `shape()`、`boundingRect()`、`mouseDoubleClickEvent()` 三处统一使用
- **优化（双端）**：组名称标签布局从右上角改为左上角
  - 更符合从左到右的阅读习惯
  - 标签位于组框左上角上方，不遮挡组内图片
- **影响文件**：
  - Windows: `Views/Canvas.py`
  - macOS: `Views/CanvasView.swift`

## 2026-02-05 周三 - 图片打组功能增强

### 下午 17:27 - 修复：组移动和双击编辑问题
- **修复问题**：移动组时图片不跟随移动
  - 原因：计算移动量时使用了 `rect().topLeft()` 而不是 `pos()`
  - `QGraphicsRectItem.ItemIsMovable` 标志会改变 `pos()` 而非 `rect()`
  - 修复后：基于 `pos()` 计算移动偏移量，移动完成后同步更新 `rect` 位置
- **修复问题**：双击组名称无法快速编辑
  - 原因：名称标签矩形检测坐标系不一致
  - 修复后：正确使用局部坐标进行点击检测
- **修复问题**：组边界检测使用 `rect()` 而非 `sceneBoundingRect()`
  - 现在正确使用场景坐标进行边界检测
- **影响文件**：
  - Windows: `Views/Canvas.py`, `Views/MainWindow.py`

### 上午 11:57 - 修复：撤销/重做快捷键冲突问题
- **问题描述**：快捷键在画布上不生效，仅顶部菜单栏可用
- **根本原因**：`QShortcut` 和菜单栏 `QAction` 同时绑定了相同的 `Ctrl+Z`/`Ctrl+Shift+Z` 快捷键，导致 Qt 报出 "Ambiguous shortcut overload" 错误
- **解决方案**：
  - 移除 `MainWindow.py` 中重复的 `QShortcut` 绑定
  - 保留菜单栏的 `QAction` 快捷键设置
  - 设置 `QAction` 的 `ShortcutContext` 为 `ApplicationShortcut`，确保在任何控件获取焦点时都能触发
- **影响文件**：`HajimiRef_win11/Views/MainWindow.py`

## A1.0.2 (2025-11-28) - 性能与体验大修
### 新增功能
*   **始终置顶**: 在设置菜单中增加了"始终置顶 (Always on Top)"选项，方便在绘画时参考。
*   **增强粘贴**: 优化了 `Ctrl+V` 逻辑，现在不仅支持粘贴截图，还支持从资源管理器直接复制文件后粘贴。

### 优化
*   **视口裁剪 (Viewport Clipping)**: 重写了图像渲染管线。现在只渲染屏幕可见区域的图像切片。
    *   *效果*: 彻底解决了图片放大倍数过大时（如 >500%）导致的严重卡顿和内存溢出问题。
*   **动态重采样 (Dynamic Resampling)**: 
    *   滚动/拖拽时使用 `Nearest Neighbor` 插值，保证 60fps 流畅度。
    *   停止操作 200ms 后自动切换为 `Lanczos/Bilinear` 插值，保证画质清晰。
*   **汉化补全**: 修复了界面中遗漏的英文文本（如 Exit, Always on Top）。
